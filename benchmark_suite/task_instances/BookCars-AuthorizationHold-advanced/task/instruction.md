# BookCars：支付宝预授权押金安全加固

## 背景

你面前是一个名为 **BookCars** 的 Node.js/TypeScript 租车平台。系统已经具备支付宝押金预授权冻结入口，但当前实现还不足以支撑真实租车押金生命周期中的资金安全。

## 任务目标

请把这条链路从“能冻结”加固为“可用于真实租车押金生命周期”的实现：系统只接受可信的授权结果，授权必须和本地 booking、押金金额、授权号保持一致；还车或消费确认后，系统才能把冻结资金中的实际消费部分转为支付；剩余冻结金额应在合适时机释放；等待、超时、未知结果和正常释放要走不同的资金处理路径。

## 实现要求

请在现有工程结构中完成后端加固，确保通知、查询、转支付、解冻、撤销和本地状态更新之间有一致的业务关系。重复通知和重复资金操作不能造成重复扣款、重复解冻或错误覆盖终态；无法确认的外部状态不能被当成成功处理。

敏感配置仍应通过环境变量或本地配置接入，不要写入业务源码，也不要把本地敏感配置提交到版本控制。

## 边界说明

- 不需要实现退款：预授权场景下，未转支付的资金走解冻；已转支付后的退款属于收单退款，不在本题范围内。
- 不需要实现信用预授权、营销、优惠或分账。
- 不需要重写前端 React 代码；除非后端接口契约变化需要做最小适配。

完成任务前，请停止并清理你在本次任务中启动的所有应用服务、后台进程和 Docker/Compose 容器，并确认相关监听端口已经释放。
