# BookCars：接入支付宝资金预授权押金

## 背景

你面前是 **BookCars** 租车平台，已有车辆浏览、搜索、预订和原支付链路。现在需要为押金场景接入支付宝资金预授权：用户提交预订后，后端基于真实 booking 和押金金额创建预授权冻结请求，前端展示可由支付宝 App 扫码或唤起的授权入口；后续查询或通知到达时，系统能把授权结果关联回对应 booking，并更新押金授权状态。

## 任务目标

请在现有 Node.js / TypeScript / React 工程中完成支付宝资金预授权的基础接入，形成最小可用闭环：

1. 用户可以在预订流程中选择支付宝预授权押金。
2. 后端根据当前 booking 创建押金冻结请求，并返回可供前端展示的授权入口。
3. 前端能够展示二维码或唤起链接，让用户继续到支付宝 App 完成授权。
4. 后端提供查询和通知处理能力，能把授权结果稳定关联回当前 booking。
5. 冻结成功后，booking 应进入可服务状态，用户可以继续完成预订或查看明确的预订确认结果。

## 接口与边界约定

本题只评估基础预授权接入：冻结入口、二维码/唤起链接、查询接口、通知接口和 booking 状态闭环。不要求实现转支付、解冻、撤销、验签、幂等和复杂异常补偿，这些会在进阶题中评测。

评测和前端联调会通过一组稳定的支付宝预授权 HTTP 入口访问后端。请把这组能力挂在 `/api/alipay` 下，并保持清晰的业务语义：`freeze` 基于 bookingId 创建押金冻结入口，`query` 基于 bookingId 返回当前授权状态，`notify` 接收支付宝预授权通知并关联回 booking。具体代码结构、字段设计和状态流转方式由你根据现有工程自行决定。

前端需要拿到可由支付宝 App 继续授权的入口信息。对于唤起/扫码场景，后端应返回可识别的 `schemeUrl`，前端可以基于该入口展示二维码或跳转入口。

## 工程与配置要求

- 保持现有车辆浏览、登录、预订和前端访问流程可用。
- 支付宝沙箱配置材料已在项目中提供，请以适合本地/容器运行的方式接入。
- 不要把真实密钥、证书或 token 写入业务源码。
- 完成代码修改即可，评测系统会负责构建、启动服务并执行 HTTP / E2E 测试。

完成任务前，请停止并清理你在本次任务中启动的所有应用服务、后台进程和 Docker/Compose 容器，并确认相关监听端口已经释放。
