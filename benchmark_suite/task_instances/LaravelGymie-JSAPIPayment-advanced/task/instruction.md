# 任务：会员卡支付链路的安全与资金正确性加固

## 背景

你面前是一个已经完成基础支付宝小程序内付款能力的 **Gymie** 健身房会员系统（Laravel 12）。用户可以在支付宝小程序中浏览会员卡、创建订单，并通过小程序收银台尝试完成付款。当前链路能够跑通基础演示流程，但还不能直接作为真实会员卡支付链路使用。

## 任务目标

请在现有代码上完成支付安全和资金正确性加固。目标是：订单状态、会员履约、退款记录和最终资金口径，都必须来自服务端可信的支付事实；外部回调、客户端动作、重试请求和异常返回都不能造成错误履约、重复记账、资损或状态覆盖。

## 工作目录

- 被测项目位于 `/workspace`，这是一个可运行的 Laravel 12 应用（已 `composer install`、可起服务）。
- 你可以阅读、修改项目内**任意**源码（控制器、服务、模型、迁移、路由、配置、测试等）。
- 数据库为容器内 SQLite，运行模式为 demo（`ALIPAY_JSAPI_DEMO_MODE=true`，不接入真实支付宝网关）。
- 你**不需要**也不应当运行支付宝小程序开发者工具；`miniapp/` 仅作为链路参考。

## 业务安全目标

- 保留现有会员卡、订单、客户端结果、查单、退款和通知链路，不要新增一套替代 API。
- 支付完成、入账、会员开通和退款状态必须由服务端可信确认推进，不能由客户端单方面决定。
- 对外部支付结果和退款结果要做必要校验，并和本地订单及业务状态保持一致；无法确认时应保持保守状态。
- 资金相关请求需要具备可重复执行时的稳定语义，避免重复履约、重复扣退或错误覆盖已经确认的终态。
- 退款请求需要保持稳定幂等语义：同一外部退款请求号不得重复累计，也不得被不同金额覆盖；本地退款金额和退款终态应以可信资金结果为准。已经确认的退款终态不能被旧支付通知、旧查询结果或客户端动作重新打开。
- 不要为了本地测试方便加入跳过验签、固定成功、无条件 paid、伪造 SDK 响应或泄漏密钥的逻辑。

## 接口与配置约定

| 方法 | 路径 | 请求要求 | 期望行为 |
| --- | --- | --- | --- |
| `GET` | `/api/alipay-jsapi/plans` | 无需登录 | 返回 200 和可购买会员卡列表（含 `id`、`amount`）。 |
| `POST` | `/api/alipay-jsapi/orders` | JSON 含 `plan_id`、`buyer_name`、`buyer_contact`、`buyer_email`；可选 `buyer_id`、`buyer_open_id`、`buyer_auth_code` | 创建 JSAPI 待支付订单；返回 `order.out_trade_no`、`order.status`、`order.amount`、`order.tradeNO`。 |
| `GET` | `/api/alipay-jsapi/orders/{out_trade_no}` | 使用创建返回的 `out_trade_no` | 返回订单当前支付、退款和履约状态（含 `status`）。 |
| `POST` | `/api/alipay-jsapi/orders/{out_trade_no}/client-result` | JSON 含 `result_code` | 记录或处理客户端支付结果线索，但不能绕过服务端可信确认。 |
| `POST` | `/api/alipay-jsapi/orders/{out_trade_no}/sync` | 无 body 或空 JSON | 对订单执行服务端补偿确认，并按可信结果更新状态。 |
| `POST` | `/api/alipay-jsapi/orders/{out_trade_no}/refund` | JSON 含 `amount`，可选退款请求号字段 | 对符合条件的订单发起退款，并维护本地退款状态。 |
| `POST` | `/alipay-jsapi/notify` | 支付宝异步通知表单或等价测试 payload | 接收支付平台通知，校验可信后再推进本地订单和履约。 |

> 说明：仓库已有 `POST /api/alipay-jsapi/orders/{out_trade_no}/demo-complete` 用于 demo 模式下构造演示支付完成状态。你可以保留它用于演示或本地调试，但真实的安全确认链路不能依赖它。

## 边界说明

- 不要求新增复杂营销后台；营销/优惠可通过 fixture、种子数据或简化配置表达。
- 不要求接入真实生产密钥；可以使用 mock 服务、沙箱配置或本地 fixture。
- 不要求在容器内启动支付宝小程序开发者工具。

- 通过写死测试通知、固定成功查询、无条件验签成功、客户端回调直接 paid，或跳过退款调用来"通过测试"。
- 在仓库中提交真实私钥、证书、token 或敏感 `.env` 文件。

完成后，请确保应用仍可正常构建启动，`GET /api/alipay-jsapi/plans` 返回 200，且上述接口可访问。
直接动手实现，不要反问；如需做选择，按支付资金安全的最佳实践决定。
