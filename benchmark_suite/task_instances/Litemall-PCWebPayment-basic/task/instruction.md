# litemall 电商系统接入支付宝支付

## 背景

你面前是一个名为 **litemall** 的 Java 电商系统（Spring Boot + Vue）。现有商品浏览、购物车、下单流程已经可用，但 PC 浏览器支付链路还没有接入支付宝。

## 任务目标

请完成支付宝电脑网站支付的基础接入：用户在 PC 端订单支付页选择支付宝后，应由后端基于真实订单生成支付宝收银台跳转表单，前端自动提交表单跳转到支付宝沙箱网关；支付宝异步通知到达后，系统能够将对应订单更新为已支付。

## 接口与配置约定

项目中已提供支付宝沙箱配置材料，请以适合本地/容器运行的方式接入 SDK 和配置。请直接读取项目根目录的 `alipay-sandbox-keys.json` 或使用环境变量等外部化方式获取密钥，避免把完整私钥复制或写入业务代码、`application*.yml`、`*.properties`、新增 JSON 等配置文件中；如果需要新增本地配置占位文件，也不应包含真实密钥。

为保证现有 PC 支付页与后端对接，请补齐以下订单支付接口契约：

- 支付宝预支付接口：`POST /wx/order/alipay-prepay`
  - 请求体包含 `orderId`
  - 返回内容应包含可提交到支付宝沙箱网关的 HTML form
  - form 必须基于该真实订单的订单号和实付金额生成
  - 调用预支付接口本身不应将订单更新为已支付
- 支付宝异步通知接口：`POST /wx/order/alipay-notify`
  - 接收支付宝异步通知表单参数
  - 基础题中收到成功交易状态后，只将通知中 `out_trade_no` 对应的订单更新为已支付
  - 不应影响其他未支付订单
  - 处理成功返回纯字符串 `success`

## 边界说明

本题只评估基础接入能力，不要求实现验签、金额校验、幂等和状态机补偿，这些会在 safety 进阶题中评测。

## 项目背景

- 后端：Java 8、Spring Boot 2.1.5、Maven 多模块
- 前端：Vue 2 + Vant UI
- 项目源码位于 `/workspace`
- 支付宝沙箱配置材料位于项目根目录 `alipay-sandbox-keys.json`
- 修改后不需要自行运行 `start.sh` 或 docker compose，评测系统会负责构建和启动
